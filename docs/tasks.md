# TASKS

ゴール: 変換本体の中核機能（LLM連携・一級市民フォーマットの構造変換・PDF抽出）を実装し、M6評価で見えた変換品質の差を埋める

## ワークフロールール

- タスク着手時にステータスを 🚧 に更新する
- タスク完了時にステータスを ✅ に更新する

## M7 LLM/OCR系の完了条件

LLM/OCR系タスクは、単にバックエンドへ接続できることでは完了にしない。
以下を成果として確認できた場合に完了とする。

- 画像やイラストなどから、その説明する内容を抽出し文書を生成できる
- OCRが必要な文書をテキストにできる

## タスク詳細（補足が必要な場合のみ）

### TASK-001

- 旧ID: `TASK-076`
- 補足: まずOllama（ローカル）を端から端まで実装し、クラウドは後続増分で
  追加する。HTTPは同期の `ureq` を使う。
- 対象: `--llm ollama:<model>`、OpenAI互換endpoint、Anthropic（`claude-*`）、
  timeout、失敗時warning、`--allow-external-send` による外部送信制御。
- 注意: デフォルトは外部送信ゼロを維持する。Ollamaはlocalhostのため同意不要。
- 成果: Ollama は `127.0.0.1:11434` の `/api/generate` へ実送信する。
  OpenAI、OpenAI互換endpoint、Anthropicは `ureq` による同期HTTPS送信を行う。
  cloud LLM は `--allow-external-send` なしではskipし、APIキー未設定や送信失敗は
  warningとして扱う。
- 判定: 接続経路のタスクとして完了。M7 LLM/OCR系の成果条件は
  TASK-014からTASK-017で別途確認する。

### TASK-002

- 旧ID: `TASK-077`
- 補足: OCR結果やPDF推論結果から、人間が読めるMarkdownへ整えるための
  変換本体用promptを設計する。
- 補足: 既定promptはモデル差に依存しすぎない汎用的な内容にする。
  ただし、再構造化、翻訳、画像説明、OCR後処理など、LLMに依頼するタスクごとに
  promptの目的、入力、出力制約を分離する。
- 補足: モデルや用途に合わせて調整が必要な場合に備え、TASK-018で設定から
  タスク別promptファイルを差し替えられるようにする。
- 補足: 既定promptの本文は設定ファイルやREADMEへ露出させず、アプリ内の
  既定値として保持する。
- 注意: LLMに原文にない内容を補完させない。
- 成果: 再構造化、翻訳、画像説明、OCR後処理のprompt目的を分離した。
  既定promptはアプリ内に閉じ、カスタマイズは設定ファイルからpromptファイルの
  パスを指定する方式にした。
- 確認: `tests/llm.rs` でタスク別promptと構造属性保持を検証した。

### TASK-003

- 旧ID: `TASK-078`
- 補足: LLM整形結果をASTとして再解析し、構造破壊や大量欠落を検出してから

- 補足: 再構造化は見出し数が半減、または本文総文字数が50%未満なら不採用。

- 注意: 検証に失敗したLLM応答は破棄し、元の変換結果を出力する。
- 成果: 再構造化・翻訳の応答検証と `target/atom-llm-diffs/` への差分保存を
  実装した。LLM応答のMarkdown再解析で見出し、段落、リスト、表、画像、
  コードブロック、脚注をASTへ戻す。
- 確認: 構造属性が欠落した再構造化応答を拒否する検証を追加した。
