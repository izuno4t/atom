# TASKS

マイルストーン: M6
ゴール: 実コーパスをディレクトリ単位で評価し、原因分類と改善優先度を判断できる状態にする

## ワークフロールール

- タスク着手時にステータスを 🚧 に更新する
- タスク完了時にステータスを ✅ に更新する
- DependsOn のタスクがすべて ✅ でないタスクには着手しない

## ステータス表記ルール

| Status | 意味 |
| ---- | ----- |
| ⏳ | 未着手、TODO |
| 🚧 | 作業中、IN_PROGRESS |
| 🧪 | 確認待ち、REVIEW |
| ✅ | 完了、DONE |
| 🚫 | 中止、CANCELLED |

## 管理方針

- M1-M5の旧タスク表は `docs/archive/tasks-m1-m5.md` にアーカイブする。
- M1-M5から残った未完了タスクは `docs/tasks-m7.md` に移動する。
- 現行 `docs/tasks.md` はM6の評価プロセスだけを管理する。
- 外部ツール比較、事前棚卸し、ディレクトリ別評価は `benchmark/` に置く。
- `evaluation/` は `atom` 本体のfixture評価と回帰検出に限定する。
- 入力ルートや実文書の絶対パスはGit管理対象へ保存しない。
- ディレクトリ別タスクにはディレクトリ名だけを保存する。
